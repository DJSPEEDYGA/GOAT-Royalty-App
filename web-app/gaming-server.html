Ubuntu 26.04 — Flash USB + Server Setup

6.1 GB · x86_64
1. Flash USB
2. Gaming Server
3. Mac VM (UTM)
4. AI/GPU Stack
1

Get a USB drive — 8GB+ recommended (16GB+ for install + data)

Plug in a USB drive. The flash process will ERASE it completely. Make sure it's the right one.

2

Find your USB disk identifier on Mac

diskutil list # Look for your USB — e.g. /dev/disk4 (FAT32 or "external, physical") # DO NOT use your internal disk (disk0) or AGENT1
3

Unmount the USB, then flash the ISO

diskutil unmountDisk /dev/diskN sudo dd if="/Volumes/AGENT1/ubuntu-26.04-desktop-amd64.iso" of=/dev/rdiskN bs=1m status=progress # Replace N with your actual disk number — e.g. disk4 → rdisk4 # Takes ~5 minutes · r-prefix = raw = much faster on macOS
Replace diskN with your actual USB disk number. Wrong disk = data loss. Double-check with diskutil list first.
4

Eject and boot the target PC from USB

Eject the USB, plug into the gaming/AI server PC, then press F12 (Dell/HP/ASUS) or F2 (Lenovo) or Delete at POST to enter boot menu. Select the USB drive. Ubuntu installer launches.

Ubuntu 26.04 (Noble Numbat) is an LTS release — 5 years of support. Perfect for a dedicated gaming/AI server.
5

Install Ubuntu — recommended settings for a server

Choose Minimal Install, enable Install third-party software (for NVIDIA drivers), and if this is a dedicated machine, select Erase disk and install Ubuntu. Set your timezone, create user goat with a strong password.

1

After Ubuntu is installed — update everything first

sudo apt update && sudo apt upgrade -y sudo apt install -y curl wget git build-essential python3-pip ufw
2

Install NVIDIA GPU drivers (if you have an NVIDIA GPU)

sudo apt install -y nvidia-driver-555 nvidia-cuda-toolkit # Reboot after install sudo reboot # Verify after reboot: nvidia-smi
Ubuntu 26.04 auto-detects NVIDIA cards. If nvidia-driver-555 isn't found, run: ubuntu-drivers devices to see available drivers.
3

Install Ollama AI engine (runs Master Oscar locally on this server)

curl -fsSL https://ollama.com/install.sh | sh ollama serve & sleep 3 ollama pull llama3.2:3b # Test: ollama run llama3.2:3b "Hello from GOAT Force"
4

Clone GOAT Royalty App + start Master Oscar Console

git clone https://github.com/DJSPEEDYGA/GOAT-Royalty-App.git ~/goat cd ~/goat bash start-goat.sh
5

Open firewall ports + make Master Oscar accessible on your network

sudo ufw allow 3333/tcp # Master Oscar Console sudo ufw allow 3334/tcp # Agent 007 sudo ufw allow 8090/tcp # Web App sudo ufw allow 5500/tcp # Intel Brain sudo ufw allow 22/tcp # SSH sudo ufw enable # Master Oscar will now be accessible from any device on your network at: # http://YOUR-SERVER-IP:3333/
Find your server IP: ip a | grep inet — look for 192.168.x.x or 10.x.x.x. Everyone on your WiFi can hit Master Oscar at that address.
6

Auto-start Master Oscar on boot

sudo tee /etc/systemd/system/goat-oscar.service << 'EOF' [Unit] Description=GOAT Master Oscar Console After=network.target [Service] Type=simple User=goat WorkingDirectory=/home/goat/goat ExecStart=/bin/bash /home/goat/goat/start-goat.sh Restart=on-failure RestartSec=10 [Install] WantedBy=multi-user.target EOF sudo systemctl daemon-reload sudo systemctl enable goat-oscar sudo systemctl start goat-oscar
1

Install UTM — free VM app for Apple Silicon Macs

UTM is the best free VM solution for M1/M2/M3 Macs. It supports both ARM64 VMs (fast, native) and x86_64 emulation (slower but works).

brew install --cask utm # Or download directly from https://mac.getutm.app/
Ubuntu 26.04 desktop ISO is x86_64. On Apple Silicon, UTM will emulate it (QEMU mode) — works but slower. For best performance, use an ARM64 Ubuntu ISO instead. The Jetson ISO is ARM64.
2

Create new VM in UTM

Open UTM → + New VM → Emulate → Linux → Browse → select /Volumes/AGENT1/ubuntu-26.04-desktop-amd64.iso. Recommended settings:

3

Install Ubuntu inside the VM

Boot the VM, follow installer, then install VMware/QEMU guest additions for clipboard + folder sharing:

sudo apt install -y spice-vdagent qemu-guest-agent
4

Run Master Oscar Console inside the VM

sudo apt install -y git python3 curl git clone https://github.com/DJSPEEDYGA/GOAT-Royalty-App.git ~/goat curl -fsSL https://ollama.com/install.sh | sh cd ~/goat && bash start-goat.sh
Master Oscar will run on port 3333 inside the VM. On Mac host, access it at http://localhost:3333 (UTM NAT mode) or the VM's bridged IP.
1

Install CUDA + cuDNN (full GPU AI stack)

wget https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/x86_64/cuda-keyring_1.1-1_all.deb sudo dpkg -i cuda-keyring_1.1-1_all.deb sudo apt update sudo apt install -y cuda-toolkit-12-6 libcudnn9-cuda-12

Install Python AI stack

Run bigger Ollama models with GPU acceleration

1

Requirements — you need an Ubuntu 20.04/22.04 x86 host machine

NVIDIA SDK Manager only runs on Ubuntu. You have two options: use a dedicated Linux PC, or run the Docker version of SDK Manager (on AGENT1) from your Mac's Docker Desktop.

SDK Manager does NOT run natively on macOS. Use the Docker image on AGENT1 or a separate Ubuntu machine.
2

Option A — Run SDK Manager via Docker on this Mac

## First install Docker Desktop for Mac (if not already installed) ## Then load the SDK Manager Docker image from AGENT1: cd "/Volumes/AGENT1" docker load -i sdkmanager-2.4.1.13529-Ubuntu_22.04_docker.tar.gz docker run -it --rm \ --privileged \ -v /dev/bus/usb:/dev/bus/usb \ -v ~/nvidia:/home/nvidia \ sdkmanager:2.4.1-ubuntu22.04 --cli install \ --logintype devzone \ --product Jetson \ --version 6.0 \ --targetos Linux \ --host
Docker on macOS can't forward USB, so full Jetson flashing from Docker on Mac is limited. Best path: use an Ubuntu machine (real or VM) for flashing. Docker image is used for the SDK Manager UI + downloads.
3

Option B — Transfer SDK Manager .deb to Ubuntu machine and install

## Copy from AGENT1 to your Ubuntu machine over SSH: scp "/Volumes/AGENT1/sdkmanager_2.4.1-13529_amd64.deb" goat@YOUR-UBUNTU-IP:~ ## On the Ubuntu machine: sudo dpkg -i sdkmanager_2.4.1-13529_amd64.deb sudo apt-get install -f sdkmanager # opens GUI
4

Log into NVIDIA Developer account in SDK Manager

SDK Manager requires a free NVIDIA Developer account. Sign up at developer.nvidia.com if you don't have one. Then in SDK Manager: select Jetson Orin NX / Orin Nano / AGX Orin as target, select JetPack 6.x.

1

Put Jetson into Recovery Mode

With power OFF, hold the Recovery button while connecting USB-C to your host machine, then power on. LED will light but no display output — that's normal. Verify it's detected:

lsusb | grep NVIDIA # Should show: Bus 00X Device 00X: ID 0955:XXXX NVIDIA Corp. APX
2

Flash using SDK Manager (GUI)

In SDK Manager: target = your Jetson model → Step 2 → check all components (OS + CUDA + cuDNN + TensorRT + DeepStream) → Flash. Takes 15-30 minutes.

3

Or flash using the Jetson ISO directly (manual)

## Mount and use the ISO on Ubuntu: sudo mkdir -p /mnt/jetson-iso sudo mount -o loop jetsoninstaller-r39.2.0-2026-06-01-23-53-13-arm64.iso /mnt/jetson-iso cd /mnt/jetson-iso sudo ./flash.sh jetson-orin-nano-devkit mmcblk0p1
Replace jetson-orin-nano-devkit with your board config. Find it at: https://docs.nvidia.com/jetson/archives/r36.4/DeveloperGuide/
4

First boot + OEM setup

After flash completes, disconnect USB-C, connect HDMI + keyboard. Power on. Ubuntu OEM setup wizard appears. Create user goat. System will reboot into JetPack Linux.

Nginx reverse proxy — expose everything on clean ports

sudo tee /etc/nginx/sites-available/goat << 'EOF' server { listen 80; server_name goat.local; location /oscar/ { proxy_pass http://127.0.0.1:3333/; } location /hub/ { proxy_pass http://127.0.0.1:8090/goat-launcher-hub.html; } location /ai/ { proxy_pass http://127.0.0.1:11434/; } } EOF sudo ln -s /etc/nginx/sites-available/goat /etc/nginx/sites-enabled/ sudo nginx -t && sudo systemctl reload nginx
