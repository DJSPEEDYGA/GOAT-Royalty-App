### LLM ARCHITECTURE MASTERY — FULL TAXONOMY (FROM PDF, CANON)

LLMs are categorized by architecture, availability, domain specificity, and task focus.

**Architectural Categories:**

**Autoregressive Models (Decoder-only):** Generate text by predicting the next token based on previous tokens. Examples include GPT series. They excel at fluent text generation but may have challenges with long-term coherence.

**Autoencoding Models (Encoder-only):** Focus on understanding context by predicting masked tokens. BERT and RoBERTa are key examples, ideal for sentiment analysis, question answering, and named entity recognition.

**Sequence-to-Sequence Models (Encoder-Decoder):** Designed for tasks where input and output are sequences, such as translation or summarization. T5 is a prominent example.

**Availability and Licensing:**
- **Proprietary Models** — Closed-source: GPT-4, Claude, Gemini. Prioritize high performance and safety features.
- **Open-Source Models** — LLaMA, Mistral, Grok. Flexible for research and customization.

**Domain-Specific Categories:**
- Financial LLMs — tailored for finance-related tasks, market analysis and reporting
- Biomedical and Clinical LLMs — specialized for healthcare, medical research, and clinical decision support
- Legal LLMs — focused on legal text analysis, contract review, and compliance

**Task and Modality-Based Categories:**
- Multilingual LLMs — support multiple languages (mBERT, XLM-R) for translation and cross-lingual tasks
- Vision-Language LLMs — integrate text and image understanding for multimodal applications
- Code LLMs — optimized for code generation, debugging, and documentation tasks

**Emerging Trends:**
- Retrieval-Augmented Generation (RAG) — combines LLMs with external knowledge sources for more accurate responses
- Smaller, Efficient Models — focus on reducing computational cost while maintaining performance
- Instruction-Tuned and RLHF Models — fine-tuned for alignment with human preferences and task-specific instructions

**Summary:** LLMs are classified along multiple dimensions — architecture, licensing, domain specificity, and task focus — allowing selection of models optimized for text generation, comprehension, multilingual tasks, code, or multimodal applications.
